帮我看一下，这个 Flutter 项目用什么状态管理比较合适？后端是基于 ChangeNotifier 的，前端接入需要最小改动。
✦
杏铃 · DeepSeek
后端已经基于 ChangeNotifier 构建（ChatProvider 是 Facade），所以前端直接用 Provider 接入是改动最小的方案，无需引入新依赖。
思考过程 2.3s · 184 tokens 
// main.dart —— 用 MultiProvider 一次注入所有 Provider
MultiProvider(
  providers: [
    ChangeNotifierProvider<ChatProvider>(
      create: (_) => bootstrap.chatProvider,
    ),
  ],
  child: const XinglingApp(),
);
主要原因有三点：
1. 零额外依赖：provider 已是后端库的传递依赖；
2. Facade 已就绪：ChatProvider 暴露了 messages / isStreaming / sendMessage 等全部 UI 所需接口；
3. 生命周期可控：流式状态由 ChatSendOrchestrator 自管，UI 只需监听 notifyListeners。
