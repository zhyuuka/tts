用户问的是 sendMessage 全流程。我需要从入口追踪到持久化： 1. ChatProvider.sendMessage 转发到 ChatSendOrchestrator 2. Orchestrator 调用 MemoryProvider.buildEnhancedPrompt 拼装上下文 3. AiService.chatStream 产出 SSE 流 4. ChatStreamingController 接收 delta，更新 streamingContent / streamingReasoning 5. 流结束后写入 storageService.saveMessagesSync 6. MemU 异步写入跨会话语义记忆
联网检索 · 3 来源
1 Flutter Provider 状态管理官方文档 flutter.dev
2 SSE 流式响应在 Dart 中的实现 dart.academy
3 Hive Box 异步写入最佳实践 medium.com

ChatProvider.sendMessage 是一个 Facade 转发，真正的工作在 ChatSendOrchestrator 中完成。整体流程分 6 步：

1. 上下文拼装

先调用 MemoryProvider.buildEnhancedPrompt，它会：

  • 注入 systemPrompt（人格设定优先）
  • 从 MemLocal 取会话内原文片段
  • 从 MemU 取跨会话语义记忆
  • 按 token 预算裁剪历史

2. 流式拉取

final stream = aiService.chatStream(messages);
await for (final chunk in stream) {
  controller.append(chunk);
}

3. 持久化

流结束后一次性写入 storageService.saveMessagesSync，避免每帧 IO。MemU 写入异步触发，不阻塞 UI。
